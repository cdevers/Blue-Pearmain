# Tags Filter Dimension — Spec (#156)

> Status: approved for implementation

## Overview

Add a tag input to the shared `_filter_bar.html` Jinja macro as a sixth filter dimension, wired to both `/library` and `/map`. The `tag` field enters `normalize_shared_filters()` so both routes normalise it the same way. A new `db.tag_names()` method supplies the datalist values.

---

## Data model

Tags are stored in two JSON array columns on the `photos` table:

- `photos_tags` — Apple Photos keywords; 2,609 distinct values; human-readable (`boston`, `concert`, `family`, …). This is the **datalist source**.
- `flickr_tags` — Flickr tags; 99,256 distinct values; includes machine tags (`exif:orientation=horizontalnormal`, `camera:make=nikoncorporation`, …). **Not used for the datalist.**

The filter logic checks **both** columns for an exact value match — consistent with the existing `_library_where()` `tag` clause. This preserves backwards compatibility with existing `?tag=` URLs and catches photos where a keyword exists on one side but not the other.

---

## New: `db.tag_names()` method

```python
def tag_names(self) -> list[str]:
    """Return distinct tag values from photos_tags JSON arrays, sorted alphabetically."""
    rows = self.conn.execute(
        "SELECT DISTINCT j.value "
        "FROM photos p, json_each(p.photos_tags) j "
        "WHERE p.photos_tags IS NOT NULL "
        "  AND p.photos_tags NOT IN ('null', '[]', '') "
        "  AND p.flickr_deleted = 0 "
        "ORDER BY j.value"
    ).fetchall()
    return [r["value"] for r in rows]
```

Pattern mirrors `person_names()`. Returns all tags (no minimum-count filter).

### `api_map_photos()` route

After the person WHERE fragment, add:

        "(EXISTS (SELECT 1 FROM json_each(p.flickr_tags) WHERE value = ?) "

## Canonical URL invariant

`tag` is the raw string value, URL-encoded. Empty string → param omitted. Whitespace stripped server-side. Case-sensitive match (consistent with existing behaviour).

## Testing

### `db.tag_names()`
- Returns a sorted list of strings.
- Excludes photos with `flickr_deleted = 1`.
- Returns `[]` when no photos have `photos_tags`.

### `normalize_shared_filters()` — tag field
- `?tag=boston` → `sf["tag"] == "boston"`.
- `?tag=+boston+` → `sf["tag"] == "boston"` (whitespace stripped).
- `?tag=` → `sf["tag"] == ""`.

### `library()` with `?tag=`
- `?tag=boston` returns only photos where `boston` appears in `photos_tags` or `flickr_tags`.
- `?tag=` (empty) returns unfiltered results.
- `?tag=no-such-tag` returns 0 photos.
- Filter chip appears when `tag` is non-empty.

### `api_map_photos()` with `?tag=`
- `?tag=boston` scopes dots/trail to photos tagged `boston`.
- `?tag=` (empty) returns all photos (subject to other filters).

### `map_view()` deep-link
- `?tag=boston` → `initial_filters["tag"] == "boston"` → macro pre-fills input → panel auto-opens.

### Cross-page nav
- Library "View on map" link with active `tag` filter carries `?tag=boston` to `/map`.
- Map `openInLibrary()` with active `tag` filter carries `?tag=boston` to `/library`.
