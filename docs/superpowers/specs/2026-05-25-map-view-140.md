# Map View — Design Spec

**Issue:** #140  
**Date:** 2026-05-25  
**Status:** Approved, awaiting implementation plan

---

## Problem

The library grid has no spatial dimension. Finding photos by location means knowing the city name — which isn't granular enough for dense home areas where thousands of photos share the same city. A map lets you drill into geographic clusters naturally, at whatever zoom level makes the distinction meaningful.

## Scope

A standalone `/map` page that plots all geotagged photos on an interactive map:

- Clustered markers at low zoom; individual markers at high zoom
- Clicking a cluster zooms and expands it (standard markercluster behaviour)
- Clicking a single marker opens a popup: thumbnail, title, date, and three links — Open photo, Flickr ↗, Show this day in library
- Nav entry at key `0`, after Albums

**Out of scope:**
- Bounding-box filter (draw a rectangle → filter library grid)
- Select-on-map for bulk operations
- Filtering the map itself by album / rating / date
- Heatmap view
- Mobile-specific touch affordances beyond what Leaflet provides by default

---

## Architecture & Data Flow

```
GET /map
    → renders map.html (empty map container + Leaflet boilerplate)
    → JS on load: fetch GET /api/map-photos
    → returns [{id, lat, lon, title, date_taken, flickr_url}]
    → JS plots MarkerClusterGroup with one marker per photo
    → clicking a marker shows popup (thumbnail via /thumb/<id>, links)
```

No persistent state. The JSON payload is fetched once per page load and kept in memory. Flickr URLs are constructed server-side (requires `flickr_id` + username from config) and included in the JSON; photos without a `flickr_id` get `flickr_url: null`.

## Backend

### New routes — `reviewer/app.py`

| Method | Path | Purpose |
|--------|------|---------|
| `GET` | `/map` | Render the map page |
| `GET` | `/api/map-photos` | JSON payload for all geotagged photos |

**`GET /map`**

Computes the initial map centre from the median of all geotagged photo coordinates, passes it to the template as `center_lat`, `center_lon`. If no geotagged photos exist, defaults to `(20.0, 0.0)` (world overview).

```python
@app.route("/map")
def map_view() -> str:
    row = db().conn.execute(
        "SELECT AVG(latitude) AS lat, AVG(longitude) AS lon "
        "FROM photos WHERE latitude IS NOT NULL AND longitude IS NOT NULL"
    ).fetchone()
    center_lat = row["lat"] if row["lat"] is not None else 20.0
    center_lon = row["lon"] if row["lon"] is not None else 0.0
    return render_template("map.html", center_lat=center_lat, center_lon=center_lon)
```

**`GET /api/map-photos`**

Returns all photos with coordinates. Includes only the columns needed for the popup; does not include `thumbnail_path` (the client uses `/thumb/<id>`).

```python
@app.route("/api/map-photos")
def api_map_photos() -> Response:
    flickr_username = _config.get("flickr", {}).get("username", "")
    rows = db().conn.execute(
        "SELECT id, latitude, longitude, photos_title, flickr_title, "
        "       date_taken, flickr_id "
        "FROM photos "
        "WHERE latitude IS NOT NULL AND longitude IS NOT NULL"
    ).fetchall()
    result = []
    for r in rows:
        title = (r["photos_title"] or r["flickr_title"] or "").strip() or "(untitled)"
        flickr_url = (
            f"https://www.flickr.com/photos/{flickr_username}/{r['flickr_id']}"
            if r["flickr_id"] and flickr_username
            else None
        )
        result.append({
            "id": r["id"],
            "lat": r["latitude"],
            "lon": r["longitude"],
            "title": title,
            "date": (r["date_taken"] or "")[:10],   # YYYY-MM-DD
            "flickr_url": flickr_url,
        })
    return jsonify(result)
```

## Frontend

### `reviewer/templates/map.html` *(new)*

Extends `base.html`. Loads Leaflet and Leaflet.markercluster from CDN (no npm, no bundler). Map fills the content area below the nav bar.

```html
{% extends "base.html" %}
{% block title %}Map — Blue Pearmain{% endblock %}

{% block extra_style %}
<link rel="stylesheet" href="https://unpkg.com/leaflet@1.9.4/dist/leaflet.css" />
<link rel="stylesheet" href="https://unpkg.com/leaflet.markercluster@1.5.3/dist/MarkerCluster.css" />
<link rel="stylesheet" href="https://unpkg.com/leaflet.markercluster@1.5.3/dist/MarkerCluster.Default.css" />
<style>
#map { height: calc(100vh - 48px); width: 100%; }   /* 48px = nav bar height */
.map-popup img { width: 120px; height: 80px; object-fit: cover; border-radius: 4px; display: block; margin-bottom: 6px; }
.map-popup .title { font-size: 13px; font-weight: 600; margin-bottom: 2px; }
.map-popup .date  { font-size: 11px; color: #888; margin-bottom: 6px; }
.map-popup .links { font-size: 12px; display: flex; gap: 10px; }
.map-popup .links a { color: #0070f3; text-decoration: none; }
.map-popup .links a:hover { text-decoration: underline; }
</style>
{% endblock %}

{% block content %}
<div id="map"></div>

<script src="https://unpkg.com/leaflet@1.9.4/dist/leaflet.js"></script>
<script src="https://unpkg.com/leaflet.markercluster@1.5.3/dist/leaflet.markercluster.js"></script>
<script>
const map = L.map('map').setView([{{ center_lat }}, {{ center_lon }}], 5);

L.tileLayer('https://{s}.tile.openstreetmap.org/{z}/{x}/{y}.png', {
  attribution: '© <a href="https://www.openstreetmap.org/copyright">OpenStreetMap</a>',
  maxZoom: 19,
}).addTo(map);

const markers = L.markerClusterGroup();

function esc(s) {
  return String(s).replace(/&/g,'&amp;').replace(/</g,'&lt;').replace(/>/g,'&gt;').replace(/"/g,'&quot;');
}

fetch('/api/map-photos')
  .then(r => r.json())
  .then(photos => {
    photos.forEach(p => {
      const marker = L.marker([p.lat, p.lon]);
      const libraryDay = p.date ? `/library?date=${p.date}` : '/library';
      let links = `<a href="/photo/${p.id}">Open photo</a>`;
      if (p.flickr_url) links += `<a href="${esc(p.flickr_url)}" target="_blank" rel="noopener">Flickr ↗</a>`;
      if (p.date)       links += `<a href="${libraryDay}">Show this day</a>`;
      marker.bindPopup(`
        <div class="map-popup">
          <img src="/thumb/${p.id}" alt="">
          <div class="title">${esc(p.title)}</div>
          <div class="date">${esc(p.date || '')}</div>
          <div class="links">${links}</div>
        </div>
      `, { maxWidth: 200 });
      markers.addLayer(marker);
    });
    map.addLayer(markers);
  })
  .catch(() => {
    /* silently fail — map is still usable, just empty */
  });
</script>
{% endblock %}
```

## Notes

**"Show this day in library" link** depends on #141 (library date filter) to actually filter the grid. Until #141 lands, the link navigates to `/library?date=YYYY-MM-DD` and the query param is silently ignored — the user sees all photos rather than just that day. Acceptable short-term behaviour; no special handling needed.

**Popup XSS:** `p.title` and `p.flickr_url` are inserted via template literals. The `esc()` helper (defined in the template JS) escapes `&`, `<`, `>`, `"` before interpolation. Applied to all user-sourced string values in the popup.

**Performance:** `idx_photos_location` (on `latitude, longitude`) already exists. At personal-library scale (≤ 100k photos), the full coordinate JSON is ≤ 10 MB and loads in well under a second on localhost. No pagination or lazy loading needed.
