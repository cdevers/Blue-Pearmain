# Spec: Photo Trails — chronological path on the map (#151)

_Status: ✓ done — shipped cab518d (2026-05-27)_

---

## Problem

The map shows all geotagged photos as independent dots. There is no way to see *how* a set of photos relates in time — whether they form a journey, a walk, a road trip, or scattered visits over years. Connecting dots chronologically turns a scatter plot into a story.

---

## Approach

Add a **Show trail** toggle to the map filter bar. When enabled (and a time filter is active), draw a `L.polyline` connecting that filter's photos in chronological order, behind the existing markers. No new API endpoint or DB changes are needed — the existing `/api/map-photos` response already includes `date` and `lat`/`lon`.

---

## Scope

**In:**
- Trail polyline drawn over the filtered photo set, sorted by `date_taken`
- Toggle in the filter bar, disabled when no time filter is selected
- Trail redrawn on every filter change
- Trail hidden when toggle is off, or when filter is cleared
- Reasonable default style (semi-transparent, directional colour)

**Out (v1):**
- Segment click / hover showing photos in that leg
- Multi-day colour coding (one colour per day)
- Animated playback along the trail
- Export as GPX

## UI changes — `reviewer/templates/map.html`

### Filter bar

Add a checkbox after the time-select, adjacent to the existing `±2 days` checkbox:

```html
<label id="map-trail-label" style="display:none;align-items:center;gap:5px">
  <input type="checkbox" id="map-trail-cb"> Show trail
</label>
```

Show/hide logic:
- `map-trail-label` is visible whenever a time pattern is selected (same condition as when `map-expand-label` is shown for holidays, but applied to *all* patterns).
- When the time pattern is cleared, uncheck and hide.

### Trail rendering

Add a module-level variable:

```js
let _trailLayer = null;  // L.polyline or null
```

New function `plotTrail(photos)`:

```js
function plotTrail(photos) {
  if (_trailLayer) { map.removeLayer(_trailLayer); _trailLayer = null; }
  if (!document.getElementById('map-trail-cb').checked) return;
  const sorted = photos
    .filter(p => p.lat != null && p.lon != null && p.date)
    .sort((a, b) => a.date < b.date ? -1 : a.date > b.date ? 1 : 0);
  if (sorted.length < 2) return;
  const latlngs = sorted.map(p => [p.lat, p.lon]);
  _trailLayer = L.polyline(latlngs, {
    color: '#e06820',
    weight: 2,
    opacity: 0.65,
    dashArray: null,
  }).addTo(map);
  _trailLayer.bringToBack();
}
```

Call `plotTrail(photos)` at the end of `plotPhotos(photos, requestId)` (after the markers are added and the count is updated).

Also call `plotTrail([])` (to clear) when the trail checkbox is unchecked:

```js
document.getElementById('map-trail-cb').addEventListener('change', () => plotTrail(_lastPhotos));
```

Store the last-fetched photo array in a module-level `let _lastPhotos = []` so the checkbox toggle can redraw without a network round-trip.

### Filter bar show/hide

Extend the `map-time-select` change handler to also show/hide `map-trail-label`:

```js
const isAnyPattern = !!this.value;
document.getElementById('map-trail-label').style.display = isAnyPattern ? 'flex' : 'none';
if (!isAnyPattern) {
  document.getElementById('map-trail-cb').checked = false;
  if (_trailLayer) { map.removeLayer(_trailLayer); _trailLayer = null; }
}
```

## Test surface

The trail logic is pure client-side JS in a Jinja template. No new Python is added, so no new pytest tests are needed. Verify manually:

1. Open `/map` without a filter → trail toggle is hidden.
2. Select a month filter → trail toggle appears.
3. Check trail → polyline appears connecting photos in date order.
4. Uncheck trail → polyline disappears; markers still present.
5. Change filter → trail redraws for new photo set.
6. Clear filter → trail removed, toggle hidden.
7. Open `/map?photo_id=N` with a filter → trail appears alongside the highlighted marker.
